## Project Overview

React SPA template for building product frontends. This is a reusable foundation — it contains no product logic, only app plumbing (auth, API client, routing, shell, components). Vite 6 + React 19 + TypeScript + Tailwind CSS v4. Node 24+ (pinned in `.nvmrc`).

## Commands

```bash
# Install dependencies
npm install

# Development server (hot reload)
npm run dev

# Production build (typecheck + vite build)
npm run build

# Preview production build
npm run preview

# Run all tests
npm test

# Run tests in watch mode
npm run test:watch

# Type checking
npm run typecheck

# Lint
npm run lint
npm run lint:fix

# Format
npm run format
npm run format:check   # used in CI

# Pre-commit checks (format, lint, typecheck, tests)
npm run precommit
```

## Architecture

### Composition Root

`src/main.tsx` is the only file that reads `import.meta.env`. Everything else receives configuration via props or constructor arguments, making all modules independently testable.

### Component Hierarchy (v0.4)

```
main.tsx (composition root — reads env, creates TokenStorage + Logger + ApiClient)
  └── App (props: apiClient, tokenStorage)
        └── ApiClientProvider (exposes apiClient via React context)
              └── AuthProvider (manages auth state + session bootstrap)
                    └── RouterProvider (TanStack Router with auth context)
                          └── __root.tsx (ErrorBoundary + Toaster + SplashScreen/Outlet)
                                ├── login.tsx → LoginPage (public-only, no shell)
                                ├── register.tsx → RegisterPage (public-only, no shell)
                                ├── _app.tsx → AppShell (sidebar + header + Outlet)
                                │     ├── _app/index.tsx → HomePage
                                │     └── _app/health.tsx → HealthPage
                                └── [404] → NotFoundPage
```

### Auth Module (`src/auth/`)

- **`AuthContext.ts`** — React context + `AuthContextValue` interface. Separated from component to satisfy `react-refresh` ESLint rule.
- **`AuthProvider.tsx`** — Provider component. Bootstraps session from refresh token on mount, delegates to ApiClient for login/register/logout. Thin React wrapper — auth logic stays in ApiClient.
- **`useAuth.ts`** — `useAuth()` hook. Consumes `AuthContext`, throws if used outside `AuthProvider`.

### Routing (`src/routes/`)

File-based routing via `@tanstack/router-plugin` Vite plugin. Route files use TanStack Router conventions:

- **`__root.tsx`** — Root layout using `createRootRouteWithContext`. Renders ErrorBoundary + Toaster (sonner), shows SplashScreen during auth bootstrap, registers `notFoundComponent`.
- **`_app.tsx`** — Pathless layout route. Auth guard (`beforeLoad`) + AppShell wrapper. All authenticated routes are children of this layout.
- **`_app/index.tsx`** — Home page (`/`). Inside app shell.
- **`_app/health.tsx`** — Health page (`/health`). Inside app shell.
- **`login.tsx`** — Public-only route (`/login`). `beforeLoad` redirects to `/` when authenticated. No app shell.
- **`register.tsx`** — Public-only route (`/register`). Same guard as login. No app shell.

The route tree (`src/routeTree.gen.ts`) is auto-generated by the Vite plugin and gitignored. It provides full type safety for `Link`, `useNavigate`, and route params.

### Page Components (`src/pages/`)

- **`SplashScreen.tsx`** — Loading indicator shown during session bootstrap.
- **`LoginPage.tsx`** — Email + password form with shadcn Card/Button/Input. Uses `useAuth().login()`.
- **`RegisterPage.tsx`** — Full registration form with shadcn components.
- **`HomePage.tsx`** — Authenticated landing with user info in a Card.
- **`HealthPage.tsx`** — API connectivity check. Uses `useApiClient()` for direct API access.
- **`NotFoundPage.tsx`** — 404 page with link to home.

### Library Modules (`src/lib/`)

- **`api-types.ts`** — All TypeScript types mapped 1:1 from elixir-api-core OpenAPI spec. Request/response interfaces, envelope types, shared types (UserSummary, AccountSummary, Role, etc.).
- **`api-error.ts`** — `ApiError` class extending `Error` with `code`, `statusCode`, `details`. Thrown by ApiClient on error responses.
- **`api-client.ts`** — Typed HTTP client. Constructor injection (`baseUrl`, `tokenStorage`, `logger`). Unwraps `{ data }` envelope, parses `{ error }` envelope, auto-refreshes on 401 with mutex to deduplicate concurrent refreshes. Methods for all auth endpoints.
- **`token-storage.ts`** — Access token in-memory (XSS mitigation), refresh token in localStorage (persistence). Methods: get/set/clear.
- **`logger.ts`** — Simple structured logger wrapping console. Level-based filtering (debug/info/warn/error/silent).
- **`utils.ts`** — `cn()` class name utility (clsx + tailwind-merge). Used by all shadcn/ui components.
- **`ApiClientContext.ts`** — React context for ApiClient instance.
- **`useApiClient.ts`** — Hook consuming ApiClientContext with provider guard.

### UI Components (`src/components/`)

- **`ErrorBoundary.tsx`** — React error boundary (class component — React limitation). Catches render errors, shows fallback with retry button.
- **`AppShell.tsx`** — Main layout: sidebar + header + content area. Manages mobile sidebar toggle state.
- **`Sidebar.tsx`** — Navigation sidebar. Desktop: fixed w-64. Mobile: overlay with backdrop. Nav items, user info footer, sign out.
- **`Header.tsx`** — Top bar with mobile hamburger toggle.
- **`ui/`** — shadcn/ui components (owned files): Button, Input, Card, Dialog, DropdownMenu, Sonner (Toaster).

### Response Envelope

The API client uses the standard envelope shared across all Tiny Inbox services:

```json
{ "data": { ... } }
{ "error": { "code": "...", "message": "...", "details": {} } }
```

## Testing

- 34 tests across 17 test files
- Uses Vitest with explicit imports (no globals) + jsdom environment
- React Testing Library for component tests + `@testing-library/user-event` for interactions
- MSW (Mock Service Worker) for network-level API mocking
- `test/setup.ts` imports jest-dom matchers, starts/stops MSW server, runs `cleanup()` between tests
- `test/mocks/handlers.ts` — MSW handlers for all auth endpoints with realistic fixtures
- `test/helpers.tsx` — `renderWithProviders` and `renderWithRouter` (auth + router + ApiClientContext) utilities
- Test file structure mirrors `src/` layout
